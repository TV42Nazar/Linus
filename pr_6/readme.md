# Завдання 2.16

Реалізувати простий налагоджувач для Linux, який дозволяє запускати дочірній процес, перехоплювати його сигнали, зчитувати стан регістрів процесора та встановлювати точку зупину (breakpoint) шляхом ін'єкції інструкції `INT 3`.

---

### 1. Встановлення необхідних інструментів

Для розробки знадобляться стандартні засоби компіляції та заголовні файли системних викликів Linux.

```bash
sudo apt update
sudo apt install build-essential
```

---

### 2. Команди компіляції та запуску

```bash
# 1. Компіляція програми налагоджувача
gcc -o simple_db debugger.c

# 2. Компіляція тестової програми, яку ми будемо досліджувати
gcc -o target_proc target_proc.c

# 3. Запуск налагоджувача
./simple_db ./target_proc
```

---

### 3. Пояснення ключових констант ptrace()

#### PTRACE_TRACEME:
- Використовується дочірнім процесом. Цей виклик повідомляє ядру, що батьківський процес буде відстежувати цей процес. Усі сигнали (крім `SIGKILL`), надіслані дочірньому процесу, змушуватимуть його зупинитися, а батьківський процес отримає сповіщення через `wait()`.

#### PTRACE_PEEKTEXT / POKETEXT:
- Дозволяють зчитувати та записувати слово даних (зазвичай 8 байт на x86_64) за певною адресою в пам'яті дочірнього процесу. Саме так реалізуються точки зупину: ми зчитуємо оригінальний байт і замінюємо його на `0xCC` (`INT 3`).

#### PTRACE_GETREGS:
- Копіює значення регістрів загального призначення дочірнього процесу в структуру `user_regs_struct`, визначену в `<sys/user.h>`. Це дозволяє бачити значення `RIP` (лічильник команд), `RAX`, `RSP` тощо.

#### PTRACE_CONT / SINGLESTEP:
- `PTRACE_CONT` відновлює виконання програми до наступного сигналу. `PTRACE_SINGLESTEP` змушує процесор виконати рівно одну машинну інструкцію і знову зупинитися.

---

## 4. Опис рішення та вихідний код

Програма працює за принципом розгалуження (`fork`). Дочірній процес активує режим трасування і запускає цільову програму. Батьківський процес очікує на сигнали та керує виконанням.

### Алгоритм роботи:

1.  Використовується `fork()`. Дитина викликає `PTRACE_TRACEME`, стаючи "підлеглою".
2.  Через `execl` запускається цільовий файл. Ядро зупиняє процес відразу після завантаження, надсилаючи `SIGTRAP` батькові.
3.  Батько використовує цикл `wait()`. Кожного разу, коли дитина зупиняється (через сигнал або завершення інструкції), ми:
    * Зчитуємо вміст регістрів через `PTRACE_GETREGS`.
    * Аналізуємо номер сигналу (`WSTOPSIG`).
    * Даємо команду на виконання наступної інструкції (`PTRACE_SINGLESTEP`).

### Реалізація Breakpoint:
Щоб встановити точку зупину на конкретну адресу `addr`:
1.  Зчитати оригінальні 8 байт: `long data = ptrace(PTRACE_PEEKTEXT, pid, addr, NULL);`.
2.  Записати `0xCC` (код `INT 3` для x86): `long trap = (data & ~0xFF) | 0xCC;`.
3.  Повернути `trap` у пам'ять: `ptrace(PTRACE_POKETEXT, pid, addr, trap);`.
4.  Коли програма дійде до цієї адреси, вона згенерує `SIGTRAP`.
