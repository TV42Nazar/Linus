# Звіт

### Тема
Дослідження, моделювання та нестандартні підходи до аналізу процесів, файлових систем, безпеки та ресурсів в Linux

### Технічні вимоги та встановлення залежностей
Для успішної компіляції та роботи всіх утиліт необхідно встановити базовий набір інструментів розробника та сервіс віддаленої реєстрації користувачів.

**Команди для підготовки системи**
```bash
sudo apt update
sudo apt install build-essential rwho -y
```

## Виконані завдання

### Задача 1
У цьому завданні реалізовано міжпроцесну взаємодію за допомогою системного виклику `popen()`. Програма створює односпрямований канал (pipe) між двома утилітами ОС. Спочатку відкривається процес `rwho` на читання, а потім процес `more` на запис. У циклі програма зчитує стандартний вивід `rwho` у буфер і передає його на стандартний ввід `more`. Це дозволяє організувати зручний посторінковий перегляд інформації про активних користувачів у системі, не створюючи тимчасових файлів на диску.

**Команди для компіляції та запуску**
```bash
gcc -Wall task1.c -o task1
sudo ./task1
```

### Задача 2
Програма виконує низькорівневий аналіз вмісту каталогу, імітуючи роботу утиліти `ls -l`. Замість виклику системної команди, використовується функція `opendir()` для відкриття поточного каталогу та `readdir()` для зчитування його елементів. Для кожного знайденого файлу викликається функція `stat()`, яка звертається до inode файлу та повертає детальну структуру метаданих. З цієї структури витягуються та форматуються права доступу (через побітові операції), кількість жорстких посилань, імена власника та групи, розмір файлу та час останньої модифікації.

**Команди для компіляції та запуску**
```bash
gcc -Wall task2.c -o task2
./task2
```

### Задача 3
Реалізовано спрощену версію утиліти `grep` для пошуку підрядка у тексті. Програма приймає два аргументи командного рядка - шукане слово та шлях до файлу. За допомогою функції `fopen()` файл відкривається у режимі читання. Далі програма порядково зчитує вміст у буфер за допомогою `fgets()` і використовує функцію `strstr()` для перевірки наявності заданого слова у поточному рядку. Якщо збіг знайдено, рядок виводиться на екран.

**Команди для компіляції та запуску**
```bash
gcc -Wall task3.c -o task3
./task3 "int" task3.c
```

### Задача 4
Написано аналог утиліти `more` для порційного виводу тексту. Програма здатна обробляти кілька файлів, переданих як аргументи. Вона відкриває кожен файл, зчитує його вміст і виводить у консоль, паралельно ведучи підрахунок рядків. Використовуючи операцію ділення за модулем, програма зупиняє виконання кожні 20 рядків і очікує вводу від користувача (натискання клавіші `Enter`) через функцію `getchar()`, після чого продовжує вивід наступного блоку тексту.

**Команди для компіляції та запуску**

### Задача 5
Розроблено програму для рекурсивного обходу дерева каталогів. Алгоритм відкриває вказану директорію і перебирає всі її елементи, ігноруючи спеціальні посилання на поточний та батьківський каталоги. За допомогою системного виклику `stat()` та макросу `S_ISDIR` програма перевіряє, чи є поточний елемент директорією. Якщо так, функція викликає саму себе, передаючи новий сформований шлях, що забезпечує повне сканування всіх вкладених рівнів файлової системи.

### Задача 6
Створено утиліту, яка виводить список лише підкаталогів у поточній папці, відсортованих за алфавітом. Замість ручного читання та сортування, програма використовує потужну бібліотечну функцію `scandir()`. Вона автоматично виділяє пам'ять під масив структур `dirent`. До `scandir()` передається кастомна функція-фільтр, яка відсіює звичайні файли, та стандартна функція `alphasort` для лексикографічного впорядкування результатів.

### Задача 7
Реалізовано утиліту для інтерактивного управління правами доступу. Програма сканує поточний каталог і за допомогою функції `strrchr()` знаходить файли з розширенням `.c` (вихідні коди на C). Для кожного такого файлу користувачеві виводиться запит на екрані. У разі підтвердження, програма зчитує поточні права доступу через `stat()` і використовує системний виклик `chmod()` з побітовим додаванням маски `S_IROTH`, що безпечно надає дозвіл на читання для групи "Інші", не змінюючи інших атрибутів файлу.

### Задача 8
Розроблено програму для безпечного інтерактивного видалення даних. Алгоритм перебирає вміст каталогу, відфільтровуючи лише регулярні файли за допомогою макросу `S_ISREG` (щоб випадково не спробувати видалити директорію). Після отримання ствердної відповіді від користувача, програма викликає функцію `unlink()`, яка видаляє ім'я файлу з файлової системи та зменшує лічильник жорстких посилань на відповідний inode, що призводить до звільнення дискового простору.

### Задача 10
Програма генерує унікальні послідовності псевдовипадкових чисел з плаваючою комою. Головною особливістю є правильна ініціалізація генератора (seed). Щоб гарантувати унікальність послідовності навіть при багаторазовому запуску скрипта протягом однієї секунди, програма використовує побітове виключне АБО (XOR) між поточним системним часом `time(NULL)` та унікальним ідентифікатором процесу `getpid()`. Числа масштабуються до заданих діапазонів шляхом ділення результату `rand()` на константу `RAND_MAX`.

**Команди для компіляції та запуску**
```bash
gcc -Wall task10.c -o task10
./task10 42.5
```

### Задача 15
Розроблено комплексний аналізатор бази користувачів та груп системи. Програма починає роботу з отримання UID поточного користувача через `getuid()`, після чого витягує його ім'я з `/etc/passwd` за допомогою `getpwuid()`. Далі використовується функція `getgrouplist()`, яка збирає ідентифікатори всіх додаткових груп, до яких входить користувач. Ітеруючи по цих групах, програма викликає `getgrgid()`, щоб отримати імена груп та масив їхніх учасників. Проходячи по цьому масиву, програма знаходить інших користувачів системи, фіксує перетини і динамічно формує структурований звіт про спільні ресурси доступу.

**Команди для компіляції та запуску**
```bash
gcc -Wall task15.c -o task15
./task15
```

### Висновок
Під час виконання практичної роботи було проведено глибоке дослідження механізмів роботи ОС Linux. Написання утиліт безпосередньо мовою C із використанням системних викликів дозволило зрозуміти процеси маршрутизації потоків даних, структурування метаданих на рівні inode та механізми забезпечення безпеки доступу до ресурсів. Отримані результати підтверджують ефективність використання POSIX API для розробки продуктивного системного програмного забезпечення.
