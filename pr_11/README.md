**Тема:** Розробка контролера подій на основі системних сигналів у ОС Linux.

### 1. Завдання
Реалізувати керуючий процес (контролер), який:
*   Створює групу дочірніх процесів.
*   Приймає асинхронні сигнали від кожного процесу.
*   Реєструє точний час надходження кожного сигналу та ідентифікатор відправника (PID).
*   Формує підсумковий графік подій (timeline) у хронологічному порядку.

### 2. Опис виконання
Для реалізації контролера використано системний виклик **sigaction** із прапорцем **SA_SIGINFO**. Це дозволяє батьківському процесу не просто фіксувати факт сигналу, а отримувати розширену інформацію про відправника через структуру `siginfo_t`.

Кожен дочірній процес імітує виконання певної роботи за допомогою випадкової затримки, після чого надсилає сигнал **SIGUSR1** батькові. Контролер зберігає ці дані в динамічній структурі. Після завершення всіх процесів виконується сортування за часовою міткою та побудова графічної візуалізації, де кожна подія відображається на часовій шкалі відносно моменту старту програми.

### 3. Команди для виконання
```bash
gcc controller.c -o controller
./controller
```

### 4. результат 


### Висновок
Реалізований контролер успішно демонструє механізм обробки асинхронних подій. Використання сигналів дозволяє мінімізувати навантаження на систему, оскільки батьківський процес не опитує дітей постійно, а реагує лише в момент надходження повідомлення. Виведення у вигляді графу наочно підтверджує коректність збору та сортування даних за часом.
