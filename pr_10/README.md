## 1. Завдання
Створити батьківський процес, який породжує декілька дочірніх процесів. Кожен дочірній процес повинен мати різну затримку виконання (`sleep`). Батьківський процес має очікувати їхнього завершення за допомогою системного виклику `wait()` та виводити їхні ідентифікатори (PID) у порядку завершення.

## 2. Опис алгоритму
1.  **Створення процесів:** За допомогою циклу та функції `fork()` створюються три дочірні процеси.
2.  **Затримка:** Кожен «нащадок» засинає на певну кількість секунд (наприклад, 3, 1 та 2 секунди).
3.  **Очікування:** Батьківський процес запускає цикл з викликом `wait()`. Ця функція призупиняє батька до моменту завершення **будь-якого** з дітей.
4.  **Результат:** Оскільки час затримки різний, процеси завершуються не в тому порядку, в якому були створені. Батько отримує PID того процесу, який «прокинувся» першим.


## 3. Команди для виконання

```bash
gcc -o my_report main.c

./my_report
```

### Висновок
В ході роботи було підтверджено, що системний виклик `wait()` повертає PID завершеного процесу в порядку його фактичного завершення, а не створення. Це дозволяє батьківському процесу ефективно збирати ресурси за дочірніми процесами, що працюють асинхронно.
